# 개인 미니 프로젝트 기획서 — 모임 일정·장소 추천기

> Day 3 기획 세션에서 작성 → 구현 완료본 기준으로 갱신.

## 1. 문제
- 누구의 어떤 불편을 해결하나? → **여러 사람이 모일 때, 모두에게 공평한 날짜·장소를 정하기가 번거롭다.** 각자 사는 곳이 달라 이동시간이 제각각이고, 날씨까지 따지면 조율이 더 어렵다.

## 2. 사용자 · 성공 기준
- 사용자: 친구·스터디·동호회 등 **여러 명이 오프라인 모임을 잡아야 하는 사람**
- 성공 기준: **참여자 거주지·후보 날짜·후보 장소를 입력하면, 이동시간·공평성·실제 날씨를 종합한 추천 순위가 표로 즉시 나온다.** (1위 조합이 강조 표시됨)

## 3. 범위
- 할 것 (필수):
  - 참여자별 거주 지역 입력 → 후보 장소까지 **이동시간 추정**(좌표 거리 기반)
  - **실제 날씨 예보 연동**(Open-Meteo, API 키 불필요) + 오프라인 시 시뮬레이션 폴백
  - (날짜 × 장소) 모든 조합을 **종합점수**로 비교해 추천 순위·1위 카드 출력
  - 가중치 슬라이더(이동시간 / 공평성 / 날씨)로 우선순위 조절
- 안 할 것 (이번엔 제외):
  - 실시간 교통 API(대중교통 실시간 경로), 캘린더 자동 연동, 로그인/저장, 모바일 앱, 그래프 시각화

## 4. 산출물
- 결과 형태: **단일 HTML 웹 앱** (`모임-일정-장소-추천기.html`) — 더블클릭으로 브라우저 실행, 설치 불필요
- 필요한 입력·데이터:
  - 입력: 참여자(이름 + 거주 지역), 후보 날짜들, 후보 장소들, 가중치
  - 데이터: 서울·수도권 14개 지역 좌표(내장), Open-Meteo 실시간 날씨 예보(좌표별)

## 5. 도구·계획
- 사용할 도구: Claude Code (+ oh-my-claudecode autopilot), Open-Meteo 무료 날씨 API
- 진행 순서 (실제 구현 흐름):
  1. 입력 UI 구성(참여자·날짜·장소·가중치)
  2. 이동시간 모델 구현(좌표 간 haversine 거리 → 대중교통 시간 환산)
  3. 점수 로직(평균 이동 + 공평성 편차 + 날씨 패널티 → 0~100점 정규화)
  4. 결과 표·1위 추천 카드 렌더링
  5. 실시간 날씨(Open-Meteo) 연동 + 오프라인 시뮬레이션 폴백, 출처 배지 표시
  6. QA(실제 API end-to-end 검증)·브라우저 실행 확인
