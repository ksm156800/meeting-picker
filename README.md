# 모임 일정·장소 추천기

참여자들의 위치, 날씨, 이동시간을 종합해 **가장 공평하고 합리적인 모임 날짜와 장소**를 추천하는 단일 파일 웹 앱입니다.

### 🌐 라이브 데모: **https://ksm156800.github.io/meeting-picker/**

## 실행 방법
- **온라인**: 위 라이브 데모 링크 접속 (설치 불필요)
- **로컬**: `index.html`(또는 `모임-일정-장소-추천기.html`)을 **더블클릭** → 브라우저에서 바로 열림

## 핵심 특징
- **실시간 날씨 예보** — [Open-Meteo](https://open-meteo.com) 무료 API 사용 (**API 키·회원가입 불필요**)
  - 후보 장소의 좌표별 실제 예보(날씨·기온·강수확률)를 가져와 추천에 반영
  - 예보 가능 범위는 약 16일 → 그 밖의 날짜이거나 **오프라인이면 시뮬레이션 값으로 자동 폴백**(데모가 깨지지 않음)
  - 각 행에 데이터 출처 배지 표시: `예보`(실시간) / `추정`(시뮬레이션)
- **이동시간**: 서울·수도권 14개 지역 좌표 간 거리(haversine)를 대중교통 시간으로 환산 (로컬 계산)
- **GUI 우선** — 참여자/날짜/장소를 클릭·선택만 하면 결과가 실시간 갱신
- **가중치 슬라이더** — 이동시간 · 공평성 · 날씨 중 무엇을 더 중시할지 직접 조절
- 열자마자 보이는 **샘플 데이터**(참여자 4명, 후보 날짜 3개, 장소 4곳) 내장

## 추천 로직
각 (날짜 × 장소) 조합마다 비용을 계산하고, 낮을수록 좋은 조합입니다.

```
비용 = w이동 × 평균이동시간
     + w공평 × 이동시간 편차(평균절대편차, MAD)
     + w날씨 × 날씨 패널티(비/더위/추위)
```

전체 조합의 비용을 0~100점으로 정규화해 **종합점수**로 표시하고, 1위 조합을 상단 카드로 강조합니다.

> **공평성 지표에 대하여** — 예전에는 편차를 `최대-최소(범위)`로 계산했는데, 이 방식은
> _3명이 같은 곳에 있고 1명만 멀리 있는_ 경우에 다수가 모인 위치를 과도하게 벌점 처리해
> 엉뚱한 중간 지점을 추천하는 문제가 있었습니다. 현재는 **평균절대편차(MAD)** 를 사용해
> "대부분이 가깝다"는 사실을 반영하므로, **소수만 이동하면 되는 다수 위치를 자연스럽게 우선**합니다.
> (2명 vs 2명처럼 진짜로 갈리는 경우에는 여전히 중간 지점을 추천합니다.)

## 사용 예시
1. 참여자의 거주 지역을 드롭다운에서 선택
2. 후보 날짜를 추가하고, 후보 장소 칩을 선택
3. 우측 표에서 추천 순위 확인 — 슬라이더를 움직이면 순위가 즉시 바뀜

## 한계 (의도된 범위)
- 실시간 **교통** API는 연동하지 않음(이동시간은 거리 기반 추정) — 다음 버전 확장 후보
- 날씨 예보는 Open-Meteo 특성상 **약 16일 이내**만 실데이터(이후는 시뮬레이션)
- 실시간 날씨를 쓰려면 **인터넷 연결 필요**(끊겨 있으면 자동으로 시뮬레이션 모드로 동작)
- 지역은 내장된 14곳 기준(좌표 추가로 손쉽게 확장 가능)
